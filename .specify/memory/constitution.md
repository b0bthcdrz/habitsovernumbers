# HON Constitution

## Core Principles

1. **No Friction**: Starting a session must be a single click. No forms, no metadata required upfront.
2. **Invisible Utility**: The app should stay out of the way. No dock icon, no intrusive windows during focus.
3. **Warm & Human Aesthetic**: Use the "Warm Paper" palette and system serif typography to make the tool feel calm and non-mechanical.
4. **Local First**: All data belongs to the user and stays on their machine.
5. **Spec-Driven Development**: All implementation must map to the approved specification and tasks.
